## 🔴 Devil's Advocate
STANCE: REJECT
The framing treats this as a render-quality problem when the rot is that no part of the pipeline can JUDGE whether a generated figure teaches the concept — on a single-user product where Alex can't vet content, an auto-generator with no teaching-correctness oracle is a machine for producing confident garbage at scale. The mergesort figure that "doesn't sort" is proof the pipeline picked the WRONG figure model (static depth-tree) for a concept whose payload is the MERGE motion; nothing noticed because families verify pixels, not pedagogy. Adding a sort-animation to mergesort by hand just relocates the reference-replication sin into the renderer. The constrained-family architecture is sound and must be kept; the missing piece is concept→figure-FIT + a generative quality gate.
KEY CONCERN: No oracle for "does this figure teach THIS concept" — the pipeline can generate infinitely, look premium, pass pixel/no-clip gates, and be pedagogically wrong, and the one human who could catch it can't. Until concept→figure fit is a machine-checkable gate, every "make it look good" effort is polishing an ungated generator.

## 🧱 First Principles
STANCE: CONDITIONAL
The true unit of quality is "this specific concept, taught this specific way" — mergesort's insight IS the recursive split-and-merge motion over time, which a static depth-tree of boxes structurally cannot carry no matter the CSS. The constrained-family architecture is sound but was MIS-FACTORED: graph-tree is a STRUCTURE family, and an algorithm-in-motion concept was fed into it, so the family CEILING (not the reskin) is why output is worse than the hand demos. Build families around what a concept needs to SHOW — the verified primitive should be a timeline/transition-sequence player whose typed data is an ordered list of array states + active spans — so "show the array sorting" is the DEFAULT a correct pipeline emits.
KEY CONCERN: The missing axis is TIME/animation — families verify spatial structure per-frame, but mergesort and most algorithm lessons are state-OVER-TIME. Without a temporal/stepped family primitive (typed data = sequence of states), the pipeline keeps emitting pedagogically-dead static figures no CSS can rescue.
