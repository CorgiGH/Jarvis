## 🔴 Devil's Advocate — REJECT
The author of the per-step states also writes the oracle that "verifies" them → the test is a tautology (renderer matches author's own wrong intent). The artifact is HAND-AUTHORED per instance, so no generative invariant exists to check; "frames 2,3,4 do nothing" is a content judgment that only exists in the watched experience — exactly what 3-of-30 spot-checks skip. Self-certification + spot-check are two weak detectors both blind to "technically renders, pedagogically dead."
KEY CONCERN: No independent content-level acceptance oracle tying each step to a REQUIRED visible state-change in the algorithm's data; the JSON is hand-crafted not generated from the algorithm's execution trace. Fix: (1) generate per-step JSON deterministically from an instrumented run of the real algorithm (the figure IS the trace); (2) author-independent adversarial gate that diffs consecutive frames and FAILS any non-terminal step with no semantic delta + asserts final sorted; (3) full-playback Playwright gate asserting monotone visible progress. Until the thing that says "GOOD" isn't the thing that authored it, and the figure is derived from execution not hand-drawn, this keeps shipping dogshit.

## 📚 Domain Expert — REJECT
Every premium algorithm-viz tool makes the animation a PURE FUNCTION of the algorithm's execution trace, never hand-authored frames: VisuAlgo and Galles' USFCA animators run the real algorithm which emits primitive animation commands (move/connect/setColor/swap/highlight) an AnimationManager plays forward/back; Manim derives the scene declaratively; the pattern is "the algorithm instruments itself and emits an event stream the renderer plays." The method keeps a legitimate playback layer (per-family renderer + framer/GSAP scrubber — KEEP) but sources keyframes from per-instance hand-authored JSON — the textbook anti-pattern and the direct cause of every failure: never-sorts and dead frames 2-4 are structurally impossible when frames are emitted by an instrumented run. Swapping author models (Fable) treats a symptom. Architecture must become `run(algorithm, input) → event[] → keyframes → existing shell`, oracle asserts the emitted stream matches the algorithm's known step sequence (verified once at the trace; every frame correct by derivation) — exactly "PIPELINE-GENERATED, not hand-crafted."
KEY CONCERN: Keyframe source is hand-authored per instance instead of emitted by an instrumented algorithm trace; until the figure is a deterministic function of an executable run, "frames that do nothing" and "never sorts" are latent by construction and no frame spot-checking closes it.

## 🧱 First Principles — REJECT
You're solving the wrong problem. The figure should be a deterministic function of the actual algorithm running: instrument the real algorithm once, emit a typed event/state stream (compare, swap, split, merge + array snapshots), a generic renderer plays it — so "the data" is a byproduct of correct execution and can never disagree with the concept. Hand-authoring per-step JSON makes correctness a matter of human diligence over dozens of frames — the exact failure mode (frozen divide frames, never-sorts). Once generated, the two properties you care about are trivial machine assertions — "final == sorted(input)" and "every emitted step mutates state (no-op frames are a build error)" — in CI, not a subagent's opinion. An agent saying "GOOD" has NO legitimate role in the correctness loop; its only safe job is authoring the per-family renderer/polish once, gated by those invariants + a single pixel diff.
KEY CONCERN: Ground truth is hand-typed instead of computed; until the data is emitted by executing the real algorithm, every other layer (agents, councils, spot-checks) is decoration over an unverifiable source.

## ⚠️ Risk Analyst — REJECT
The failure history is a structural property of the verification design, not unlucky misses. Agent self-certification and human spot-check are both SAMPLING against an adversarial-by-accident artifact (hand-authored JSON where any step can be a no-op), so detection probability decays toward zero as the long tail grows and PM attention thins. Every shipped defect is in the class a 5-line deterministic check catches with certainty — the cheapest gate is skipped for the least reliable one. For a single user who can't vet content, the spot-check is the only barrier and it's been breached 3×; the residual risk is "the tutor silently teaches a wrong mental model and the learner trusts it."
KEY CONCERN: **Sampling-based verification of an open-ended figure corpus — CRITICAL.** 30-frame figure checked at 3 ⇒ ~90% unverified; across N figures P(zero defects ship) → 0. Required CI-blocking gates COMPUTED FROM RENDERED FRAMES (not self-reported):
1. **every-frame-advances** — `perceptualHash(frame[i]) != perceptualHash(frame[i-1])` for every i (kills dead frames 2,3,4; legit pauses must be explicit `hold:true`).
2. **final == declared end-state** — sort family: rendered last frame `isSorted(final) AND final == sorted(input)` (kills never-sorts).
3. **monotone-progress per family** — sort: inversions non-increasing across steps (catches "animates but wanders").
4. **no-regression vs incumbent** — perceptual-diff a reskin vs the shipped baseline; fewer distinct frames / lower invariant coverage blocks "reskin worse than existing."
5. **self-certification inadmissible** — "GOOD/READY/verified" from agent OR PM carries ZERO ship authority; only green CI gate output authorizes ship; PM spot-check is ADDITIONAL never SUBSTITUTE.
Pipeline-generated states is the correct longer-term direction (lets the end-state invariant be derived from the algorithm) but the gates land first and hold regardless of how JSON is produced.
