## 🔴 Devil's Advocate (round 1: REJECT)

AGENT: Devil's Advocate
STANCE: CONDITIONAL
REASONING: W4b's RAG injection materially addresses my round-1 hallucination concern — grounded prompts with `[Source: ...]` and "do not invent details" instruction, plus the visible `[STUB]` prefix as a degradation signal, is the right shape. But the case-insensitive *exact-string* match is a worse trap than I called out: PS lectures will have `## Lanțuri Markov` while the seed concept catalog (English-heavy) likely says `## Markov chains` — every PS quiz fires STUB despite the user having ingested the exact PDF, training the user to ignore the STUB prefix as "always on" noise within 2-3 sessions, after which it becomes invisible warning theater. Compounding flaw: 8KB hard-cut "until next `## ` or EOF" will guillotine mid-derivation in dense lecture notes (PS measure-theory sections routinely run 12-15KB under one heading), and the prompt instruction "do not invent details outside the provided text" cannot stop the LLM from completing a half-truncated proof — it will hallucinate the tail with high confidence because the visible context looks authoritative.
KEY CONCERN: Heading-match must be normalized (lowercase + accent-strip + EN/RO synonym table) AND truncation must end at the last complete sentence/paragraph boundary with a visible `[truncated — N more KB available via [[read]]]` tail marker — without both, STUB becomes ignored noise and grounded quizzes hallucinate truncated tails.

## 🧱 First Principles (round 1: REJECT)

AGENT: First Principles
STANCE: REJECT
REASONING: W4b is theater. The RAG lookup is keyed on `## {ConceptName}` exact-string match against headings that don't exist in the user's PDFs — seed_concepts.sh was authored from textbook taxonomy ("Process scheduling", "Markov chains"), while the actual PDFs use Romanian UAIC headings ("Planificare procese", "Lanțuri Markov" or worse, lecturer-specific phrasings). The match rate will be near-zero even for SO+RC and PS where ingest runs; PA/ALO/POO get 100% [STUB] because no ingest is configured. So in steady state the bot still asks textbook-framed trivia for ~180 of 187 concepts, and the [STUB] tag just makes the failure legible instead of fixing it. The correct architecture is inversion: extract concepts FROM the ingested PDFs (heading + first-paragraph as the canonical concept record), let FSRS schedule those, and delete seed_concepts.sh entirely — the catalog is downstream of the source material, not parallel to it.
KEY CONCERN: Seed catalog and PDF heading text were authored independently with no alignment check; W4b's exact-string match will miss on the vast majority of concepts, so the user still gets a trivia bot for everything except the handful of accidentally-matching headings — fix is to derive the catalog from PDF extraction, not bolt grounding onto a hardcoded list.

## ⚠️ Risk Analyst (round 1: CONDITIONAL)

AGENT: Risk Analyst
STANCE: CONDITIONAL
REASONING: [HIGH] The `_extras/` directory filter as described is semantically OK if implemented correctly — `Files.walk` is a stream of all descendants, so to "skip the directory traversal entirely" the spec must use `Files.walkFileTree` with `preVisitDirectory` returning `SKIP_SUBTREE`, OR filter the stream by checking ANY component startsWith `_` (not just top-level name). More importantly, [CRITICAL] the W4b body-extraction regex will silently fall back to STUB on the majority of concepts because seed names ("FCFS scheduling", "page replacement") will not literally match PDF chapter heading text ("## Chapter 3 — Process Scheduling"); the spec needs either fuzzy/contains-match with normalization OR an upfront name-mapping pass at ingest time, otherwise W4b ships as theater. [HIGH] Verified in source: `KnowledgeFsrs.recordReview` reads prior state then calls `appendTo` non-atomically — concurrent `/api/chat` quiz-grade calls on the same concept WILL double-grade (TOCTOU between `stateFor`/`lastReviewTs` and `appendTo`); the per-write `synchronized(LOCK)` on `appendTo` does not close this window. [MEDIUM] Smoke-after-deploy with no staging means W5 failures serve broken code to phone.
KEY CONCERN: W4b STUB-rate will be 70-90% in practice unless seed concept names are pre-mapped to actual PDF heading text; without a normalization/mapping pass, "RAG grounding" is functionally absent and the spec's claim of "8KB context per quiz" is misleading.
