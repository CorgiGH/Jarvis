## 🔴 Devil's Advocate

AGENT: Devil's Advocate
STANCE: REJECT
REASONING: W1 ships 187 disembodied `## headings` with zero body text into archival/, then W3 builds a "catchup" feature that picks "untouched concepts" from that catalog and feeds them to a quiz prompt that has no grounding text. The LLM will be asked to quiz the user on a name like "Algoritmul lui Dijkstra" with literally nothing but the heading — it will hallucinate the content from training-data priors, and the user (preparing for a UAIC-specific exam graded by UAIC-specific professors with UAIC-specific notation conventions) will study confidently against drift. W4 is explicitly carved out as "expands concept NAMES, not LLM grounding — quiz prompt unchanged, deferred to follow-up," which means the spec knowingly ships a catchup loop that quizzes against empty headings for the entire 6-hour build, and the acceptance test ([[quiz: PA]] returning *something*) cannot detect this because hallucinated quizzes look fluent. This is the same class of failure as council 1778078829 — last-N retrieval dominated by junk content — except this time the junk is generative rather than retrieval-based, and the user can't even tell it's junk because they're using the system to learn material they don't yet know. Shipping a confident-sounding wrong-answer engine to a finals-prep user 13 days before a no-restanță deadline is worse than shipping nothing.
KEY CONCERN: W3 catchup + quiz quizzes against bare `## headings` with no body text; W4 explicitly defers quiz grounding; user studies hallucinated UAIC content under finals pressure with no way to detect drift. Either land W4 grounding (full pipeline: extract → chunk → inject into quiz prompt) in the same ship, or cut W3 catchup entirely and ship only schedule + FSRS reviews (which quiz against user-authored `[[remember]]` notes that have actual body text).

## 📚 Domain Expert

AGENT: Domain Expert
STANCE: CONDITIONAL
REASONING: The architecture genuinely matches established patterns — FSRS-5-lite over Ebbinghaus is the correct call (FSRS-4.5/5 outperforms SM-2 on benchmark.spaced-repetition.com by ~10-15% on log-loss; "lite" = published defaults, no per-user weight optimization, which is right for n=1 cold-start with zero history), and the multi-day planner with daily caps mirrors Anki's `New cards/day` + `Reviews/day` deck-options idiom and Khan Academy's "daily mission" cap. However, **two divergences from standard practice are real failure modes**: (1) The spec keeps W3's review queue cap at 5/day but the existing planner already ships `MAX_REVIEW_ITEMS = 8` — Anki's default is 200/day reviews + 20/day new precisely because lowering review cap below FSRS-due count creates an exponentially-growing backlog ("review hell" — well-documented on r/Anki). With 187 seed concepts and FSRS-5 R<0.7 thresholds, daily due will exceed 5 within ~14 days post-seed. (2) Quiz from concept name only, no body grounding is the textbook RemNote/Mochi failure mode and the exact reason NotebookLM, Khoj, and MemGPT all use chunk-grounded retrieval (RAG) for question generation — naming a concept ("Bayes' theorem") gives the LLM zero scaffolding for *which* formulation, *which* notation, *which* course's emphasis. The W4 ingest writes body chunks to `archival/{subject}/extras/{slug}.md` but the quiz prompt explicitly stays unchanged — that's leaving the entire value of W4 on the floor.
KEY CONCERN: Quiz prompt UNCHANGED in W4 is the showstopper — you're paying the cost of PDF ingest (W4) without claiming the benefit. Either extend the `[[grade]]`/quiz prompt to load `archival/{subject}/extras/{slug}.md` body as RAG context when present (5-line change, this is what Khoj/MemGPT/NotebookLM all do), or cut W4 entirely and ship W1-W3 only. Current spec is the worst of both worlds: ingest tax paid, ingest dividend untaken. Secondary: review cap 5/day will produce backlog within 2 weeks of seeding 187 concepts — either uncap reviews like Anki does (only cap *new*) or add an explicit "review debt" rationale line in the renderer so the user knows when to bulk-clear.

## ⚙️ Pragmatist

AGENT: Pragmatist
STANCE: CONDITIONAL
REASONING: 5-6h all-in is the wrong shape for someone 13 days from a no-restanță deadline. W4 is the expensive part (~2h Python+pypdf+regex+idempotency+per-subject section detection across 11 PDFs / 1.2GB) and by the spec's own admission it doesn't change quiz quality this session — the prompt still only sees concept names, so W4 buys you a longer hallucination menu, not better questions. W1+W2+W3+W5 (~3h) ships [[catchup: N]] end-to-end with seeded data and tests, which is the actual user-visible win; that's the 90%-at-50%-cost cut. Bank W4 as a follow-up commit when concept bodies actually get injected into the quiz prompt — otherwise you're paying ingestion tax for no behavioral delta. The W1+W2-only path is too thin: no catchup tool means no proof the multi-day planner works, and the next session has to re-page-in the same context to finish W3 anyway, which is worse than just doing it now.
KEY CONCERN: W4 ships ingestion infra whose output the quiz prompt doesn't consume yet — pure cost, zero quality lift this session. Defer it.

## 🧱 First Principles

AGENT: First Principles
STANCE: REJECT
REASONING: The real problem is "user has 13 days to PS HW + 24 to finals across 5 subjects with existing notes/labs/lectures already on disk and needs to actually study them." The spec solves a different problem: "populate an FSRS scheduler with 187 hardcoded concept names so a quiz-trivia loop has something to ask about." A from-scratch design would start by ingesting the user's actual coursework (Desktop/SO/, lab solutions, past exam attempts, the very Second-brain wiki and study-guide platform already listed in memory) and grounding every quiz/explanation in that material — not by shell-scripting a generic concept list and having the LLM hallucinate questions from training data plus a heading string. W4 admits this gap and then explicitly does not close it: "quiz prompt UNCHANGED — W4 only adds names to catalog." That is simulating study, not enabling it. Worse, with finals 24 days out and zero ingested material, 5-6h spent on multi-day planners and placeholder exam dates is the wrong allocation — the binding constraint is "bot doesn't know your material," and every workstream except a real W4 is downstream of fixing that.
KEY CONCERN: W1+W3+W5 are all built on top of a quiz engine that generates questions from concept names alone; without retrieval-grounded quizzing against the user's actual UAIC material, this ships a trivia bot dressed as a study companion — and the user will catch that on first quiz where the bot asks about "Markov chains" using textbook framing instead of Olariu's lecture framing.
