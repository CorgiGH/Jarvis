## 🔴 Devil's Advocate
STANCE: REJECT
REASONING: The "served data_json===oracle(seed)" CI gate is self-defeating the moment you acknowledge the callout field exists. You cannot have an equality gate over a string that contains pedagogically-generated prose — so the actual gate degrades to "oracle-derived STATE fields match" while the callout field floats free, ungated. Green CI will pass a figure whose callout says the wrong thing, says it in English when the app is Romanian, or omits the Phase-C pedagogy beat. You have not retired the correctness problem — you split it into a gated half (state) and an ungated half (callout) and called it solved. The single-algorithm-impl claim is also false: callout generation is a SECOND authoring pass, so you still have a hand-authored artifact that can diverge from the state it narrates.
KEY CONCERN: The CI gate cannot cover the callout field, so it certifies only half the figure; the prose that teaches remains hand-authored and self-certified — the exact structural flaw B2 was meant to eliminate.

## 📚 Domain Expert
STANCE: CONDITIONAL
REASONING: B2 matches the VisuAlgo / Galles / algorithm-visualizer.org canonical pattern precisely: instrument once, emit a state stream, golden-file the keyframes, gate CI on golden-file equality. It is also how Manim's render pass works — the rendered artifact is served, not the live computation. The harness becoming a CI gate (served JSON === oracle(seed)) is standard snapshot/golden-file testing. The one structural flaw is the callout field: callouts are NL prose, so a byte-equality gate on the full data_json breaks the moment a callout is edited — the classic "snapshot tests that include human prose rot immediately" trap (Storybook/Manim subtitles).
KEY CONCERN: Split the gate — (1) structural/semantic gate on algorithm-derived state fields === oracle(seed), (2) a separate NL-quality gate on callout fields (non-empty, language-check, no placeholder) — never a monolithic data_json===oracle equality.

## ⚙️ Pragmatist
STANCE: CONDITIONAL
REASONING: B2 is right-sized for a single-dev local-first tutor (6 families, a handful of algos); the codegen-artifact pattern (generated file checked in, CI asserts it matches the generator) is battle-tested and cheap to debug — you diff the artifact. The real maintenance cost is the callout field: NL prose inside the same checked-in blob as deterministic keyframes means every callout edit triggers a full codegen re-run or forces a callout-exempt carve-out — design it upfront. Three-month hidden complexity: the codegen script drifts from the harness (different import path / seed normalization), CI passes because both are wrong the same way, a new family breaks silently.
KEY CONCERN: The equality gate must exclude or separately gate `callout` from day one — else the first pedagogy edit reds CI and the reflex is to weaken the gate rather than fix the architecture.

## 🧱 First Principles
STANCE: CONDITIONAL
REASONING: The irreducible goal: a student-visible figure must be a faithful rendering of the algorithm's actual execution. The minimum trustworthy system needs two INDEPENDENT things: (1) a reference execution KNOWN correct (not just self-consistent), and (2) a gate proving the figure derives from it. B2 collapses both into one artifact — the oracle both writes and checks data_json, so "data_json===oracle(seed)" is a pure tautology: it proves codegen ran, not that the algorithm is correct. The mergesort failure would pass this gate identically. The irreducible independent check is "does the oracle's OUTPUT equal the correct postcondition for this seed" (e.g. finalFrame.array === knownCorrectSort(input)), independent of the figure pipeline.
KEY CONCERN: Same oracle generating + verifying is a closed loop that can be consistently wrong; the "is the algorithm correct" check must be a separate, independently-authored assertion on the postcondition + invariants, not derived from the oracle.

## ⚠️ Risk Analyst
STANCE: CONDITIONAL
REASONING: The self-consistency trap is dominant: a buggy oracle (wrong pivot, off-by-one merge) passes data_json===oracle(seed) trivially because artifact + reference are computed by the same buggy code. The semantic gates (final==sorted, every-frame-advances) are the only independent check, but a subtly-wrong algorithm can still reach a sorted final via a non-canonical path — teaching wrong intermediate steps, exactly the exam-prep blast radius. The callout field is the second critical gap: NL-authored, semantically independent; a re-derive gate invalidates it unless exempted, and that exemption is a coverage hole. Stale-artifact drift is real but recoverable (CI blocks on mismatch). Non-determinism risk LOW given seed-deterministic execution.
KEY CONCERN: CRITICAL — oracle self-consistency: a buggy algorithm passes data_json===oracle(seed) AND can pass final==sorted (right result, wrong method), silently teaching wrong intermediate logic. Mitigation: a SECOND independently-authored reference oracle OR property-based per-frame invariants cross-checked against a trusted stdlib, not just the final state.

CORE FINDING:
B2's DELIVERY MECHANISM is correct and industry-standard (golden-file codegen from an instrumented oracle, frozen signature, families+scrubber unchanged — VisuAlgo/Galles/Manim). But B2 as framed has two fixable holes that the council surfaced unanimously: (1) a monolithic `data_json===oracle(seed)` byte-equality gate cannot cover the generated `callout` prose and would certify only the state half; (2) the same oracle generating AND checking data_json makes that equality a TAUTOLOGY — it proves codegen ran, not that the algorithm is correct, and `final==sorted` alone is necessary-not-sufficient (a wrong method reaching a sorted final passes). Neither A (Kotlin backend — two-impl drift, zero code) nor C (hybrid — frozen-sig churn, zero code) was preferred by any agent; B (TS single source) stands once the gate is fixed.

AGENT CONSENSUS: 1 REJECT, 4 CONDITIONAL — 0 flagged. The REJECT (callout coverage) is fully absorbed by the gate-split fix; no agent endorsed A or C.

KEY ISSUES:
- TAUTOLOGY / oracle self-consistency (CRITICAL, First Principles + Risk): correctness cannot come from re-running the generator. Need INDEPENDENT per-frame invariants + postcondition checked against a TRUSTED reference (trusted stdlib sort / property-based), authored independently of the oracle's step logic. `final==sorted` is insufficient; per-frame structural invariants (multiset conserved — already in the harness; plus the algorithm's defining invariant, e.g. selection-sort prefix sorted & ≤ suffix) catch wrong-method-right-result.
- CALLOUT/PROSE SPLIT (HIGH, all 5): the gate must be FIELD-SCOPED — algorithm-derived STATE fields re-derived === oracle (PROOF, Layer-1); `callout`/prose fields gated SEPARATELY by the language gate + Phase-C pedagogy (ESTIMATOR, Layer-2). Never byte-equality on the whole blob. (This matches the plan's own A1 note + two-layer model.)
- DRIFT (MEDIUM, Pragmatist): the codegen script and the invariant harness must share ONE oracle module + ONE seed-normalization path, or both go wrong together. Single import, single source.

RECOMMENDED PATH:
Adopt B2 with the gate redesigned as THREE field-scoped legs, not one equality:
  (1) STATE re-derivation (PROOF, L1): regenerate the algorithm-derived state fields from oracle(seed); served data_json state === regenerated state.
  (2) INDEPENDENT INVARIANTS (PROOF, L1): per-frame structural invariants + postcondition asserted against a TRUSTED INDEPENDENT reference (not the oracle): final==knownCorrectSort(input), multiset conserved every frame, every-frame-advances over the state vector, monotone-progress, plus the algorithm's defining per-frame invariant. This is the leg that kills the tautology and the never-sorts class.
  (3) CALLOUT/PROSE (ESTIMATOR, L2): `callout` fields excluded from (1)/(2); gated by the language gate (RO) + the Phase-C pedagogy estimator; never byte-pinned.
The oracle + seed-normalization live in ONE shared module imported by both the codegen and the invariant harness. Build A0 (commit the WIP families/oracles) first. Mergesort re-bind (off graph-tree onto a real seq-array merge with final==sorted) rides leg (2).

CONFIDENCE: 8/10
What would raise it: confirming the existing arrayMergeTrace harness already authors its per-frame invariants INDEPENDENTLY of the oracle's step generator (if invariants are derived from the same step list, leg (2) needs a genuinely separate reference — the open question for A1). Lower bound held by the not-yet-seen internals of the untracked WIP oracles.
