# Council review — 1781797515

**Problem:** The tutor generates questions (lesson beats + drills + mock-exam items) but has no machine gate that checks a generated QUESTION is well-posed — unique-or-bounded correct answer, answer present, no second defensible reading. The cuts-vs-depth bug fell through this hole. LLM-as-judge is banned for correctness.
**Proposed approach:** A 3-tier gate routed by answer type — Tier 1 computable→non-LLM oracle (compiler/SymPy/simulator) hard gate + output canonicalization for tie/float non-uniqueness; Tier 2 open-class→structural-property check or decompositional partial-credit; Tier 3 tutor-invented conceptual MCQ→minimize (mirror real exam forms) + correct-by-construction + CoVe blind re-answer flag + honest "neverificat"/withhold. Division: machines verify truth, user approves experience, user never judges correctness.
**Project context:** jarvis-kotlin one-pass digestion teaching engine. Kotlin backend (grader chain, ContentValidator, trust-net), TS/React viz, Python SymPy subprocess, Node gates, SQLite. Constraints: NO paid LLM APIs; user cannot vet content (no-oracle-inversion); LLM-judge unreliable for correctness (binding law); quality sole primary; deploy-last; SymPy is PA-only today; corpus ~6 PA KCs, others near-empty. Prior councils (1781781668, 1781789477) rated earlier fixes FLAWED; concluded correct-by-construction > post-hoc validator.

## 🔴 Devil's Advocate
AGENT: Devil's Advocate
STANCE: REJECT
REASONING: The brief admits (finding #4) the cuts-vs-depth bug class is tutor-invented conceptual MCQ — routed to Tier 3. Tier 3's only correctness mechanism is "correct-by-construction + CoVe blind re-answer flag." CoVe is an LLM technique, and the brief's OWN cited research (2026-06-18-trustworthy-ai-verification-research.md:26) states CoVe "degrades on deep reasoning; NOT a substitute for an oracle"; the SESSION-81 binding law says LLM-as-judge cannot be trusted for correctness. So the gate for the precise class that produced the bug is anchored to a leg its own evidence disqualifies; cuts-vs-depth sails through under a "guarded" label. Worse, deploy-first subject is POO, and NonLlmLegs.kt:204 routes everything but PA to NoneLeg (zero non-LLM coverage) — POO's MCQ class is all Tier 3, so the only deployed subject gets zero deterministic well-posedness checking while Tiers 1–2 harden PA, which is not on the critical path. Same coverage/false-confidence inversion the prior council (1781789477, FLAWED, conf 9) already rejected, re-dressed in three tiers.
KEY CONCERN: The bug class the gate is built to catch (linguistic two-reading ambiguity in tutor-invented MCQ) lands in Tier 3, whose correctness leg (CoVe) is an LLM the project's own law and research forbid as a correctness oracle — so cuts-vs-depth is NOT computationally caught; meanwhile the deploy-first subject (POO) has no non-LLM leg at all, making "questions are now well-posedness-gated" a green label over near-zero-coverage reality.

## 📚 Domain Expert
AGENT: Domain Expert
STANCE: CONDITIONAL
REASONING: The type-routed spine is the right architecture and maps onto known-good practice: Tier 1 is exactly the PrairieLearn "compute the key in code so it can't drift from the prompt" model plus the autograder-runs-reference-solution pattern, hardened with best-of-N (RLVR inference-time) — correct, approved. The Tier 1 uniqueness requirement is right but UNDERSPECIFIED: "no distractor also satisfies the constraint" is a model-counting / uniqueness query that should be discharged by an SMT solver (Z3, or mcp-solver over Z3/MiniZinc), not by enumerating distractors, because the genuine cuts-vs-depth failure was two satisfying readings, and count(models)==1 over the encoded constraint is the only sound test. Tier 2's structural-property checks are property-based-testing / STACK-style proof-object checking done correctly. The weak point is Tier 3: CoVe is explicitly classified in your own research as "NOT a substitute for an oracle" yet is leaned on as the flag for the linguistic-ambiguity class — the one place the design re-imports LLM-judgment for correctness.
KEY CONCERN: The linguistic-ambiguity class (the actual cuts-vs-depth bug) has NO deterministic oracle in this design. The standards-honest resolution both your research and the prior council point to: don't DETECT linguistic ambiguity post-hoc (CoVe will rationalize past it), ELIMINATE it by construction — minimize tutor-invented MCQ, derive every option+key from ONE verified source span, and for the irreducible residue use honest neverificat/withhold, not a CoVe green light. Demote CoVe to advisory/shadow (consistent with the standing "vision leg advisory-only" rule), never a gate; gate the "well-posed" label to per-subject real coverage so POO/structural items aren't silently relabeled validated.

## ⚙️ Pragmatist
AGENT: Pragmatist
STANCE: CONDITIONAL
REASONING: Build cost is far lower than the brief implies because the type-routed spine already exists: GraderRouting.chainFor/classify is the 3-tier router, ExecutionGrader.run() already returns raw oracle output separated from grading, NumericOracleGrader.matches()/gradeSymbolic() is the wired SymPy/numeric oracle, RubricGrader's structural matchers are the Tier-2 mechanism. Honest net-new work is small: an author-time harness pointing these legs at a generated question instead of a student answer, a uniqueness check (run oracle against each option), CoVe-flag + neverificat/withhold plumbing, and the one genuinely fiddly piece — semantic output canonicalization (sort-before-compare for ties/unordered), since ExecutionGrader.normalize() only handles whitespace today. The 3-tier framing is mostly a re-use story. The disproportion is reach-vs-corpus: Tier 1's oracle is PA-only SymPy and the live corpus is ~6 PA KCs with ALO/PS/POO near-empty, so building Tier 3's full conceptual-MCQ apparatus now is gold-plating against content that doesn't exist.
KEY CONCERN: Sequence by where content actually lives. Condition: build Tier 1 + Tier 2 first as a thin author-time wrapper over the existing legs (replace the LLM self-solve/critic in DrillGenerator.generate() with a real-oracle pass + uniqueness + canonicalization); ship Tier 3 only as the cheap neverificat/withhold default until non-PA subjects have committed KCs; defer CoVe + full correct-by-construction MCQ machinery until a corpus exercises it. Canonicalization is NOT always decidable (NP-equivalence of two programs' outputs in general), so scope it to bounded real cases — sorted-output / unordered-container / float-tolerance — and DEFER-to-withhold rather than guess when it can't canonicalize.

## 🧱 First Principles
AGENT: First Principles
STANCE: CONDITIONAL
REASONING: Stripped of framing, the goal is not "validate questions" — it is "never let an item whose keyed answer is not provably the intended-unique answer reach the learner." From scratch I would not build a question-validator for the dominant case; I would make the generator correct-by-construction: derive the answer from the source span by running the oracle, then synthesize the stem and distractors FROM that derived answer — so a key↔stem mismatch is structurally impossible, not caught after the fact. The proposal's Tier 1 is exactly this and I endorse it; but its phrasing as a "gate" that "asserts the keyed answer matches" reveals a residual post-hoc framing the last two councils flagged, and it never closes the gap that actually bit (cuts-vs-depth is linguistic — the oracle returns green because key=3 is a valid integer-in-options). My from-scratch design and the proposal converge on the spine (machine-first, type-routed, residue withheld) and the law (§1.2), so this is not the wrong problem — but Tier 3, where the bug lives, is still "minimize + flag + hope," not a construction that makes the bug impossible.
KEY CONCERN: The proposal does not eliminate the class of bug that motivated it. No tier offers a computational test for "second defensible reading exists" — Tier 1's oracle is blind (the key is a valid value), Tier 3 leans on CoVe (an LLM, forbidden for correctness). The first-principles-sound move: make tutor-invented conceptual MCQ NOT EXIST for graded surfaces (generate from a single verified span so options are the oracle's true-answer plus construction-disjoint distractors, never paraphrase-MCQ), and route everything else to free-form + structural-property grading — which is what real exams actually are (PA reex 2024: 100% free-form, Q1's ambiguity is the objective, machine-distinguishable only by an author-set "ambiguous-by-design" flag, never inferred). Approve the spine; require Tier 3 graded MCQ be construction-eliminated rather than CoVe-flagged, and "ambiguous-by-design vs drafting-error" be an explicit authored intent bit.

## ⚠️ Risk Analyst
AGENT: Risk Analyst
STANCE: CONDITIONAL
REASONING: The fail-closed spine is sound and already implemented (ran=false is never coerced to a pass in NonLlmLegs.kt:61/68 and ExecutionGrader defers on missing toolchain), and CoVe stays a flag-not-gate so seed (a) does not re-import the banned judge — provided a CoVe divergence can only WITHHOLD, never approve. But two seeds are real and computationally demonstrable. Output canonicalization (seed b) is NOT always decidable: ExecutionGrader.normalize() (line 231) only does whitespace/CRLF, and the POO freq-map tie-order case has no canonical form unless the intended ordering is pinned in the stem — canonicalizing on the wrong assumption silently flips a true ambiguity into a false PASS, worse than no gate. Oracle coverage (seed c/e) is PA-only today (nonLlmLegFor line 204), so Tier 1's hard gate covers almost nothing — POO, the only real MCQ subject and home of the code-output well-posedness bugs, falls to NoneLeg and degrades straight to Tier 3 residue, meaning the gate withholds rather than verifies exactly where it's most needed.
KEY CONCERN (ranked):
- CRITICAL — canonicalization is not decidable for the ambiguity it claims to catch. Tie-order/unordered output has no canonical form unless the tie-break is pinned in the stem. Invert the rule: when output ordering is underspecified, FAIL-CLOSED and demand the stem pin the tie-break (or property-check the multiset); NEVER canonicalize-then-compare on an assumed order. Mitigation: a non-LLM "output-determinism probe" — run the reference under ≥2 distinct container/hash seeds or input permutations; if stdout differs, the question is ill-posed → withhold.
- HIGH — Tier 1 hard gate is vacuous outside PA today; starvation is the default. With POO/ALO/PS/SO-RC → NoneLeg, best-of-N over an absent oracle discards all candidates → coverage gap; "minimize tutor-invented MCQ" compounds it on low-content subjects. Mitigation: gate the rollout — a subject without a runnable oracle gets items WITHHELD-and-labelled neverificat, never served-as-verified; track a per-subject "verifiable fraction" metric so starvation is visible; wire it before any subject ships.
