## 🔴 Devil's Advocate

```
AGENT: Devil's Advocate
STANCE: CONDITIONAL

PICKS:
1) Separate recentByImportance(n) returning List<ConversationEntry> (NOT ChatMessage). HIGH
2) Skip tool entirely for v1. MEDIUM
3) Pure importance + chronological re-sort, no rank-blend math at this scale. HIGH
4) ADD new "# Salient prior turns" block in buildChatContext as PLAIN TEXT.
   Leave recentAsChatMessages() chat-array call EXACTLY AS-IS. HIGH
5) Treat null as 0.4; DO NOT backfill. MEDIUM

FIX-BEFORE-SHIP:
- Regression test: recent(n) byte-identical before/after this change
- Cap input pool to readAll().takeLast(500) — don't make O(file_size) worse
- Final emit chronologically sorted by (ts, seq) — LLM reads top-down
- Document null=0.4 means pre-1.2 rows tie; tie-break by recency
```

## 📚 Domain Expert

```
AGENT: Domain Expert
STANCE: CONDITIONAL
KEY CONCERN: Don't invent a formula. Generative-Agents (Park et al. 2023) /
    LangChain TimeWeightedVectorStoreRetriever already solved this.

PICKS (named precedent):
1) Separate recentByImportance(n) — Letta MemGPT splits recall_memory_search
   from recall_memory (recency-paged); call sites differ.
2) New [[recall_salient]] tool — Mem0 splits search() (semantic) from
   get_all() (recency). Conflating with [[recall]] (semantic) breaks model.
3) Generative-Agents triad (recency+importance, no relevance term):
   score = exp(-λ * hours_since) + importance
   λ = ln(2)/24 (24h half-life). LangChain TimeWeightedVectorStoreRetriever
   default decay_rate ≈ 0.01 ≈ ln(2)/24h.
4) Parametrize buildChatContext with mode: RECENT|SALIENT (default RECENT).
   Letta MemGPTConfig.recall_memory_type does this.
5) Null importance → 0.4 (below-median, not skipped). Mem0 backfills missing
   with neutral default; LangChain defaults to 1.0 (full weight) which
   would over-prioritize old rows here.
```

## SYNTHESIS / VERDICT

**STATUS: CONDITIONAL — ship the design below.**

Convergences:
- Q5 null = 0.4 (3-of-3 align)
- Q4 buildChatContext: Devil's load-bearing insight wins — **DO NOT** reorder chat-replay messages (breaks user/assistant pairing). Render as plain text in system prompt instead.
- Q2 tool: 2-of-3 say no new tool — internal use only.

Picks the synthesis honors:
1) **Separate `recentByImportance(n)` returning `List<ConversationEntry>`** (Devil + Domain). Different return type than `recent` makes a flag-on-existing awkward; separate function is clearer.
2) **No new ChatTool.** Internal feed only.
3) **Generative-Agents `score = exp(-λ·hours) + importance`**, λ = ln(2)/24h (Domain's named precedent).
4) **NEW plain-text "## Salient prior turns" block** in `buildChatContext` system prompt. **`recentAsChatMessages()` chat-array call STAYS UNCHANGED** (Devil's critical guard).
5) **Null importance → 0.4** in the scoring view. No backfill.

Fix-before-ship (Devil):
- Cap candidate pool to `readAll().takeLast(500)`.
- Chronologically re-sort top-N by `(ts, seq)` before emit.
- Regression test for `recent(n)` shape parity.

LOC estimate: ~30 LOC Conversations + ~10 LOC WebMain + ~80 LOC tests. Within budget.
