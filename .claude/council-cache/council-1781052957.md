# Council review — 1781052957

**Problem:** Define the ideal FORM of a single-concept interactive lecture for an ADHD visual learner (loves 3B1B). We have a working, learner-approved single-concept animation (MergeSort divide+merge, eased, object-permanent, in-place, narrated). Open question = the structure of the FULL lecture that contains it.
**Proposed approach:** A lecture = a guided STEP-THROUGH of ~4–6 focused full-screen BEATS (Brilliant.org-style, advance with Next; not a scroll, not one canvas, not just the animation): ① Hook+Predict → ② Animated demo (viz centerpiece) → ③ Explanation → ④ Drill (productive failure) → ⑤ optional Check. FORM fixed; engine picks which beats + order per concept+mastery; animation is one beat.
**Project context:** Interactive CS study-tutor, single ADHD visual learner; React + brutalist design; working declarative auto-layout viz engine (graph/tree/chart/timeline/grid) with eased animation; engine decides pedagogy per concept+mastery; productive-failure pedagogy LOCKED (attempt → worked example → definition); figures must be faithful to lecture source; content Romanian; lessons must be cheaply authorable at scale.
**Timestamp:** 2026-06-10 (epoch 1781052957)

## 🔴 Devil's Advocate
AGENT: Devil's Advocate
STANCE: CONDITIONAL
REASONING: The fatal tension is between "FORM is FIXED" and "the engine decides which beats and their order/adaptivity" — contradictory: if the engine reorders/omits, the form is NOT fixed (it's an adaptive sequence engine masquerading as a deck); if fixed, you've hard-coded a sequence wrong for many concepts (e.g. Big-O has no natural HOOK+PREDICT — nothing to predict before you have the vocabulary). More critically: unmedicated ADHD + a "Next button to advance" is the worst interaction model — it demands sustained volition with no pull/reward/surprise; Brilliant is built for motivated self-improvers, this learner needs content to PULL him forward. The DRILL-after-EXPLANATION order also violates the productive-failure lock (attempt must come BEFORE the worked example).
KEY CONCERN: Beat order (HOOK/PREDICT → DEMO → EXPLAIN → DRILL) puts practice AFTER explanation — the opposite of the locked productive-failure sequence; the form as specified is pedagogically self-contradictory.

## 📚 Domain Expert
AGENT: Domain Expert
STANCE: CONDITIONAL
REASONING: The beat-based step-through matches Brilliant.org's "interactive article" format (fixed narrative spine, one idea per screen, reader-paced) — the most validated model for self-directed STEM learners with attention-regulation difficulty. Duolingo's "one interaction per screen" micro-commitment applies. predict→demo→explain→drill maps to Bret Victor's "first show then name" + Sweller's worked-example effect. The one divergence: 3B1B's power is CONTINUOUS narrated transformation — the animation runs THROUGH the explanation with voice-synced morphing; isolating the viz to Beat 2 risks "watch then forget" where the learner can't reconnect static Beat-3 text to the motion. Khan Academy's documented failure is exactly this: animation in one pane, explanation in another, retention drops.
KEY CONCERN: Beat 3 (EXPLANATION) must not be a static text card referring to an animation the learner can no longer see — either the figure persists/loops with callouts, or Beat 3 is interactive replay-with-annotation; otherwise it ships the Khan disconnection bug in a prettier wrapper.

## ⚙️ Pragmatist
AGENT: Pragmatist
STANCE: CONDITIONAL
REASONING: The fixed 5-beat form is right — it collapses authoring from "design a lesson" to "fill slots," the only way a solo builder survives 50+ concepts. The engine's 5-family viz coverage is the load-bearing bet: fits a family → demo is data-only and cheap; doesn't fit → back to bespoke code (the MergeSort trap at scale). The drill beat is the highest-cost slot (needs pedagogically-meaningful misconceptions, not noise) and will compound without a generation strategy. "Engine picks beat order per mastery" adds runtime complexity not needed until 10+ concepts are live — premature now.
KEY CONCERN: The animated demo is only cheap if the engine covers the concept; every exception is a hand-build and exceptions accumulate — need a hard rule that a non-mapping concept degrades gracefully to a static figure rather than blocking the lecture.

## 🧱 First Principles
AGENT: First Principles
STANCE: CONDITIONAL
REASONING: The raw goal is one outcome: the student can RETRIEVE and APPLY the concept under pressure, not just recognize it after watching. From scratch, the minimum unit producing that is a tight predict→see→err→explain→act cognitive cycle that must COMPLETE before moving on — not a screen, a cycle. A fixed 4–6 beat form is a reasonable container for one pass and Brilliant supports it, but it becomes wrong the moment "fixed" lets the student passively slide through beats 1–3 to the drill without ever committing a prediction or being wrong. "Learning when stuck" is served only if the engine detects "he slid through" and forces re-entry to predict — which requires a GATE, not just a sequence.
KEY CONCERN: The form has no explicit commitment gate — an un-bypassable predict step before the animation — without which the entire productive-failure theory collapses into passive watching.

## ⚠️ Risk Analyst
AGENT: Risk Analyst
STANCE: CONDITIONAL
REASONING: CRITICAL — the "next-next-next" skip is the primary ADHD failure mode: a fixed linear form with no forced interaction gate lets him click through demo + drill without processing, producing a session that FEELS complete but deposits nothing; the form can't distinguish "understood" from "moved on." HIGH — engine auto-picking beats per concept silently force-fits relational concepts (complexity comparisons, graph algorithms needing two simultaneous structures) into one figure per beat, with no signal the figure is inadequate until a downstream exam fails — a week of silent mislearning before detection. MEDIUM — sameness fatigue: identical structure across 20+ concepts becomes a skip-cue; he pattern-matches "this is a lecture" and disengages before beat 2.
KEY CONCERN: No forced-interaction gate between beats — the form permits passive click-through without verifying engagement, defeating the pedagogy for an ADHD learner who exploits any frictionless path.

The step-through-beats form is the RIGHT direction — it matches Brilliant.org's validated interactive-article model and is the only authoring-scalable shape for a solo builder. But do NOT build it as specified: three flaws would break it, and all five clean lenses converged on them. The two that matter most are unanimous — the form has no forced commitment GATE (fatal for an ADHD learner who will click through and learn nothing), and the animation is isolated to one beat instead of running THROUGH the explanation (the documented Khan-Academy "watch-then-forget" disconnection bug).

AGENT CONSENSUS: 5 CONDITIONAL, 0 flagged — unanimous "right shape, wrong details."

KEY ISSUES:
- **No commitment gate (CRITICAL, raised by First Principles + Risk Analyst):** the predict/attempt must be un-bypassable and engagement verified, or the form degrades into passive next-next-next watching that deposits nothing.
- **Beat order inverts the locked pedagogy:** practice/attempt must come BEFORE the explanation/worked-example (productive failure), not after. The proposed ①predict→②demo→③explain→④drill is backwards on the drill.
- **Animation must FUSE with the explanation, not sit in its own beat:** the figure persists/replays with callouts while the "why" is told (3B1B binding), else Beat 3 is a static card referring to motion the learner can no longer see.
- **Secondary:** graceful degradation to a static figure when a concept doesn't map to a viz family (Pragmatist); vary structure across lessons to avoid the sameness skip-cue (Risk Analyst); resolve "fixed form" to mean fixed beat-VOCABULARY with adaptive selection, not a contradiction.

RECOMMENDED PATH:
Keep the step-through deck. Rebuild the beat spec as: ① HOOK + un-skippable PREDICT (the gate — can't advance without committing) → ② ATTEMPT/struggle (productive failure, before any answer) → ③ ANIMATED REVEAL fused with the explanation (the figure stays live and is annotated as the "why" is narrated — not a separate text card) → ④ NAME/definition + invariant → ⑤ CHECK/transfer. Engine picks which beats are present + adapts order; the beat vocabulary is fixed. Add a hard degradation rule (no viz family → static figure, never block). Build ONE concept end-to-end in this corrected form and test it on the learner before generalizing.
