## SYNTHESIS / VERDICT — CONDITIONAL, ship with these picks

### Convergences
- Path: polling, WorkManager, Notification (not snackbar). All 3 agree.
- 401 graceful degrade: re-auth Notification + back off. All 3 agree.
- Default-on (auto-enable when authed): Domain + Pragmatist 2-of-3.

### Picks
1) **Interval: 15 min PeriodicWorkRequest** + foreground OneTimeWorkRequest on `LaunchedEffect(authToken)`. Devil + Domain (2-1 over Pragmatist's 30). Flood concern handled at #2.
2) **API shape: `GET /api/signals?since=<ISO8601>&limit=10`**, oldest-first. Hard limit cap = anti-flood (Pragmatist's mechanism, slightly higher cap to avoid losing signals). NO cursor token, NO pagination — append-only JSONL is ts-ordered. Devil's `?since=<id>` rejected: signal IDs are sha256 hashes (not monotonic ints).
3) **Surface: system Notification.** `setVisibility(VISIBILITY_PRIVATE)` on lockscreen → title-only ("jarvis: signal"); expanded body shows snippet. Combines Devil's privacy guard + Pragmatist's "user feels value". No in-app snackbar (Pragmatist YAGNI).
4) **Ack: SKIP server endpoint.** Client `lastSeenTs` in DataStore is the ack. Server `/api/signals` stays read-only. Pragmatist 2-1.
5) **Battery: NetworkType.CONNECTED only.** No battery-not-low (Domain's suggestion too restrictive for finals; Doze already throttles). Pragmatist wins.
6) **Auth-failure: 401 → post sticky "Re-auth jarvis" Notification, back off WorkManager interval to 4h, do not silent-retry.** All 3 align.
7) **Default state: ON when authToken non-blank.** Wire enqueue inside the existing `LaunchedEffect(authToken)`. No new settings UI. Domain + Pragmatist 2-1.

### Server-side PII guard (post-impl 1778164815 echo)
The salient-block fix taught us that ANY codepath shipping conversation/ctx-model content to a third-party (here: Android Notification) must run `CoreMemory.scanTextForPii` before emit. Apply to `signal.snippet` server-side at `/api/signals` response build. Drop tripping rows with WARN. Same shape as the buildChatContext fix in commit `3c96bcb`.
