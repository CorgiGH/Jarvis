## 🔴 Devil's Advocate — CONDITIONAL on A → hybrid(A+B)
Verified: protection 404; frame-gate hard-blocking but fires only on push/PR with no required-status (reds AFTER the bad commit lands); zero deploy steps. Hole in pure-A: a failing CI run does NOT stop the push — "Alex watches CI" is the only barrier, and for ADHD/build-everything a red check on an already-merged commit is exactly the signal that gets deferred and accreted on. The fast self-test already exists headless, so B's hook is near-free and converts core-regression from "noticed eventually" to "blocked at push" with zero PR tax.
KEY CONCERN: under pure A nothing BLOCKS a regression; broken figures silently ACCUMULATE on main and Phase-J protection only catches the last one at deploy — debugging a pile, not a diff.

## 📚 Domain Expert — CONDITIONAL on A → hybrid(A+B)
Full protection now (C) is premature: PR-and-wait on a single committer for a gate protecting an artifact no user can see — "don't gate harder than the blast radius." But "rely on a human watching CI" IS the anti-pattern the gate exists to fix (merge figure shipped 4 dead frames past human eyes), so A alone leaves the vigilance hole. Shift-left: a pre-push hook running the fast deterministic check (B) catches it in ~2s with no ceremony. A+B compose (B=local fast vitest half; A's CI=authoritative browser route-walk half B can't run); binding C belongs at the Phase-J deploy gate.
KEY CONCERN: the pre-push hook must not silently become the ONLY enforcement — it misses the browser route-walk half; its output must state it covers only the local subset and that CI frame-gate + Phase-J required-check are authoritative.

## ⚙️ Pragmatist — APPROVE A → hybrid(A + advisory B)
Measured: full frame-gate job ~93s wall (gate step 50s + npm ci + chromium) — far too slow per push; C forces PR-and-wait + that round-trip on every change, buys nothing (no auto-deploy). B costs measured ~3.8s cold (79ms/7 tests) + one hook file. Honest ceiling: the self-test proves the gate MACHINERY (decoder/diff/classifier on fixtures) is alive — it does NOT render today's figures, so it catches "someone broke the gate," not "this commit's figure shipped dead"; --no-verify bypasses it.
KEY CONCERN: don't let the 3.8s hook masquerade as "main is figure-safe" — only the ~93s CI frame-gate renders this push's figures; CI RED must stay authoritative, hook stays advisory.

## 🧱 First Principles — APPROVE A → hybrid(A+B), defer C to Phase J
Enforcement only matters where code can reach a user; verified the only such moment is a future MANUAL deploy (zero deploy triggers; served bundle moves only at Phase J). So the load-bearing invariant is "frame-gate GREEN at deploy time," not "every intermediate main commit GREEN" — C gates commits no student sees (net-negative friction now); A satisfies it by deferring the block to Phase J; B is orthogonal (fast local pre-flight, must stay advisory/bypassable). Cleaner framing: separate the SIGNAL (always-on RED, keep) from the BLOCK (bind only at the user-reachable boundary, Phase J).
KEY CONCERN: at Phase J the deferred block must ACTUALLY be wired — make "promote frame-gate to a required check / pre-deploy assertion" an explicit checked Phase-J task, not an implicit assumption, or enforcement is silently dropped exactly when it starts to matter.

## ⚠️ Risk Analyst — CONDITIONAL on A → hybrid(A + deploy.sh gate-coupling fix)
Verified + the key finding: `tools/deploy.sh` ships whatever is in the local tree — runs only `gradle :test :installDist`, scp's dist, verifies via `/healthz` 200 + `/login` title — it NEVER invokes frame-conjunction-gate.mjs / vitest / Playwright and NEVER checks HEAD passed CI. And the master-plan Phase-J EXIT GATE is a paint/4xx live-probe — a frozen-render-over-live-state figure PAINTS fine and returns 200, so the live probe does NOT catch the exact class the frame-gate guards. Under A a red frame-gate on main pre-Phase-J has near-zero blast radius (no auto-deploy), BUT "enforce-at-deploy" is currently a NO-OP because deploy isn't gate-coupled. B is --no-verify-bypassable + absent on fresh clone; C invites friction-driven disable.
KEY CONCERN: the Phase-J deploy ships the local tree with no frame-gate/CI-pass precondition, and the live-probe EXIT GATE passes frozen figures — so A is only safe if `deploy.sh` gains a HARD precondition (HEAD's frame-gate CI green, or run the gate locally) before the scp.
