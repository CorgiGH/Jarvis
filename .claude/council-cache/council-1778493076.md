## 🔴 Devil's Advocate

STANCE: REJECT
REASONING: Phase A doubles down on the exact pipeline that just produced four hallucinated problems and is poised to produce twelve hallucinated cards — the free-tier OpenRouter LLM is not a reliable extractor of glyph-decomposed Romanian math, and bolting a grounding_check substring + per-subject keyword regex on top is security theater the model can trivially satisfy while still generating wrong content (e.g. a drill saying "use `rlaplace` to derive MLE" passes both gates and is still off-task). The real defect is upstream: PDFBox is corrupting the input before any prompt sees it, and "tell the LLM to reconstruct the diacritics" is asking a free-tier model to do OCR-by-vibes on broken combining marks during the same call where it must also classify action verbs and preserve parameters. Worse, Phase A never specifies a DB migration to wipe stale hand-seeded rows, so even if the new pipeline works perfectly, Alex's live workspace still shows the hallucinated MLE drill until somebody manually re-runs `/reprep` on every task — and you've burned 2-3 of your ~21 days of runway on a regen pipeline that competes with hand-curating ~10-15 problem sets directly. With finals on Jun 1, the dominant strategy is hand-curated subject templates + LLM-fill only the numeric data, not a grounded-gen architecture whose validator the model writes around.
KEY CONCERN: Time-to-finals vs. failure-mode asymmetry — a hand-curated template per Tema gets Alex correct cards in hours with zero hallucination risk; Phase A spends 2-3 days building a validator the LLM can game, on top of a PDF extractor that's already corrupting the ground truth, with no migration plan for the stale rows already poisoning the live UI.

## 📚 Domain Expert

STANCE: CONDITIONAL
REASONING: The proposed pipeline reinvents weak versions of two solved problems. (1) Structured output: a post-hoc Kotlin validator with retry is strictly worse than OpenRouter-forwarded `response_format={type:"json_schema", strict:true}` (OpenAI Aug-2024 Structured Outputs, Anthropic tool-use) which gives compiler-grade type guarantees at decode time — Outlines/Guidance/lm-format-enforcer literature (Willard & Louf 2023, "Efficient Guided Generation") shows constrained decoding is ~100% schema-valid where prompt-and-pray hovers around 70-85%. Use schema for shape, validator only for *semantic* constraints (verbatim_quote substring, subject keyword). (2) Few-shot for glyph reconstruction is the wrong tool — Brown et al. 2020 and Liu et al. 2023 ("Pre-train, Prompt, Predict" survey) show few-shot helps task framing, but glyph artifacts are an OCR/extraction problem, not an inference problem; Marker (Datalab, 2024) and Mistral OCR (2025) destroy PDFBox on Romanian diacritics + LaTeX math, and Adobe Extract API or unstructured.io's hi_res model with a Tesseract-Romanian fallback would eliminate the reconstruction step entirely. Khan Academy's Khanmigo, Magicschool.ai, and Carnegie Learning's LiveHint do NOT do free-form 4-card LLM gen against a raw PDF — they use curriculum-aligned templates with LLM filling slots (Khanmigo's published architecture, 2024 OpenAI case study; CMU's Tutor Authoring Tools / CTAT literature, Aleven et al.), which directly supports template-per-subject over free-form. On grounding-check: SelfCheckGPT (Manakul et al. 2023) and Anthropic's citation tool show source-quoting reduces hallucination meaningfully *only when verified against retrieved text* — a self-quoted field the LLM also generated is trivially gamed; you MUST verify the quote is a substring of `pdfTextRaw` post-gen (which the plan does — keep that, it's the load-bearing bit). Finally, Q7 is the real issue: A1 asks for an R script calling `rlaplace` and plotting; the 4-card DEFINITION/WORKED/DRILL/CHECK structure is a *concept-learning* scaffold (Bloom's, ICAP framework, Chi 2014) and will not produce executable R. Replit's Code Tutor and Codecademy's CodeBytes embed a REPL; without one, Slice 2 polishes the explanation layer while Alex still can't submit homework.
KEY CONCERN: Replace PDFBox with Marker or Mistral OCR (eliminates the glyph-reconstruction prompt-engineering problem entirely), use OpenRouter's `response_format=json_schema` for shape validation, and add a 5th "EXECUTE" card with an embedded R/Python REPL — otherwise Slice 2 ships a more accurate explanation of homework Alex still cannot submit.
