## 2. Locked Cross-Cutting Contracts (the anti-refactor core — FROZEN before any build)

### 2.1 Data model — every new table/column

**Migration discipline (B1 — CORRECTED):** Exposed `.default()` is a **client-side INSERT default only** — it does NOT emit a SQL `DEFAULT` clause, and `createMissingTablesAndColumns` emits `ALTER TABLE ADD COLUMN … VARCHAR(16)` with **no** SQL default, so on SQLite every existing row is filled `NULL`. Therefore: every new column is `NULLABLE` **and** every new non-null-intent column gets an **explicit post-ALTER backfill `UPDATE … WHERE col IS NULL`** in the same boot transaction, immediately after `createMissingTablesAndColumns`. Phrase the contract as: **"Exposed client-side default + explicit post-ALTER backfill UPDATE."** Schema is NEVER ALTERed before a verified off-box `tools/db-backup` dump asserting `MIN_EXPECTED_CARDS=800` (M-DB). Migration wrapped in try/catch → on a failed ALTER, log the failing column + auto off-box dump + abort non-zero; recovery = restore-from-backup (SQLite ALTER is **not** transactional) (M-PARTIAL).

### 2.4 Verification state machine (§2.5, LOCKED)
```
UNVERIFIED --(curate-tutor emits claims, sets PENDING)--> PENDING
PENDING --(all legs agree + roundtrip + non-LLM pass)--> FAITHFUL
PENDING --(legs collapse to one family / definitional w/ no gold span / non-LLM leg == NONE)--> UNCERTAIN
PENDING --(legs disagree / roundtrip fails / span re-locate fails / a leg threw)--> FAILED-or-UNCERTAIN
FAITHFUL --(REPORT WRONG)--> PENDING
```
Invariants: **no path reaches FAITHFUL without BOTH a non-LLM-leg-pass AND families-agree.** A family-collapse (both legs same configured family, H3) ⇒ UNCERTAIN. A thrown leg (429/relay-asleep, H4) ⇒ UNCERTAIN, never a mid-txn crash. No auto-clear from student attempts. **Status persists in `kc_verification_status` (B8), never in YAML.** **FAIL-LOUD (H5):** the runner distinguishes "disagreed" from "never ran" (family unprovisioned) — never silently FAITHFUL, never silently green when a family was absent.
